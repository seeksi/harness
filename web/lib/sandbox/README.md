# lib/sandbox — safe agent sandbox

The sole owner of the agent-isolation layer: the primitives that launch a HEADLESS Claude Code build agent under defence-in-depth confinement. The single public entrypoint is **`runAgentInSandbox(opts)`** (`opts: { prompt, allowedTools?, resourceLimits?, model?, cwd, sessionId?, spawnFn? }` → `{ exitCode, sessionId, usage, audit }`). It enforces every isolation guarantee at once: a server-built validated `claude` argv (never raw client strings, `shell:false`), an **exact tool allowlist** (`DEFAULT_TOOLS = Read,Edit,Write,Grep,Glob` — Bash and free-form predicates are unreachable, and `allowedTools` is validated against it so anything outside is rejected), **`--strict-mcp-config`** always on with no `--mcp-config` so **zero MCP servers** load (the Max-plan account-synced connectors, global, and project `.mcp.json` are all ignored), a **credential-free minimal env** allowlist (no `ANTHROPIC_API_KEY`, no daemon secrets; `PATH` pinned), a **privilege drop** to a dedicated non-root low-priv OS account via `sudo -n -H -u <user> -- <abs-claude>` (required in production; the agent's only writable area is the worktrees allow-dir, the real FS jail), **worktree confinement** (cwd pinned to the lane's minted entry under `../<repo>.worktrees/<slug>`, lexically and via realpath so a symlink can't escape — see `worktree.ts`), a **default-OFF gate** (refuses unless `ENABLE_AGENT_EXEC=1`), a process-group **timeout** that SIGTERM→SIGKILLs the whole tree, and an **audit record** that carries a curated safe summary (lane/model/session) but NEVER the prompt or session token — returned to the caller as the substrate's observability. `resourceLimits` (`memoryMax`/`tasksMax`/`cpuQuota`/`cpuSeconds`/`wallMs`) is plumbed to the agent-exec wrapper via `SANDBOX_MEM_MAX`/`SANDBOX_TASKS_MAX`/`SANDBOX_CPU_QUOTA`/`SANDBOX_CPU_SECONDS` env vars (and `wallMs` becomes the in-process spawn timeout); the wrapper (built in the later provisioner chunk) reads these and applies them via cgroup/ulimit, defaulting to its current hardcoded caps when a var is unset, so today's behavior is unchanged. The threat-model doc, one-command provisioner, and conformance test are a separate later chunk.
